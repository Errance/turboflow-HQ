## 已收敛决策

| 决策 | 结论 | 影响 |
| --- | --- | --- |
| 用户体系 | 采用渠道独立身份域，不采用共享用户池 | 渠道用户不可跨登录 TF 主站，TF 主站用户不可跨登录渠道站 |
| 资金体系 | 采用渠道独立资金域 | 余额、充值、提现、cashbook、盈亏按 `fund_domain` 隔离 |
| 跨域资金 | 默认禁止 | 不允许 TF 充值从渠道站提现，也不允许渠道充值从 TF 主站提现 |
| 产品版本 | 统一升级到 v2.3 | 事件合约与足球预测市场均建立 v2.3 文档目录 |
| 事件合约池子 | v2.3 仍 100% 走 TF 公共池 | 仅做渠道入口、身份 / 资金隔离、markup 与收入对账 |
| 足球预测市场池子 | v2.3 引入分销池开关 | OFF 走 TF 公共池；ON 走渠道分销池 |
| L1:L2 比例拆分 | 不纳入 v2.3 | 同一笔订单不做拆单 / 合单比例分流 |

## P0 开放问题

1. **markup 口径**
   - 选项 A：乘法因子，`channel_quote = base_quote * (1 + markup_rate)`。
   - 选项 B：odds 点数加法。
   - 建议：优先采用乘法因子，便于复用现有 price factor / slippage / buffer-rate 机制。

2. **分销池不健康时的行为**
   - 选项 A：自动回退 TF 公共池。
   - 选项 B：直接拒单 / 暂停该渠道市场。
   - 需要确认：渠道方是否接受自动回退，以及回退后风险与收入归属如何展示。

3. **markup 收入结算路径**
   - 选项 A：直接结算到渠道提成地址。
   - 选项 B：先进入 TF 分销中转地址，再按周期清分。
   - 需要确认：财务、审计、链上转账成本与合规要求。

4. **同一外部身份跨域注册策略**
   - 选项 A：允许同一钱包 / 邮箱在不同 `identity_domain` 下创建独立账户。
   - 选项 B：检测到重复身份后提示更换凭证。
   - 需要确认：Privy / KYC / 客服处理成本。

5. **合约层 channel metadata**
   - 选项 A：仅后端订单与 pool config 保存 `channel_id`。
   - 选项 B：合约 / chain event 明确写入 channel pool metadata。
   - 需要确认：链上审计、对账与后续多渠道扩展需求。

## P1 开放问题

1. 渠道法币入金 provider、KYC / AML 主体、Merchant of Record 由谁承担？
2. 渠道出金是否首期上线，还是先只做充值与交易余额？
3. 分销池路由开关配置粒度是渠道级、产品级、市场级，还是三者都支持？
4. markup 配置粒度是渠道级、产品级、市场级，还是 pair / market 级？
5. 配置变更生效策略是立即生效，还是下一个市场 / 下一个结算周期生效？
6. 渠道收入报表首期需要按日、按市场、按用户还是按订单明细导出？
