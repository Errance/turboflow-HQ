## 5. WebSocket（实时赔率数据流）

SIG 通过 WebSocket **持续推送赔率数据帧**。这是**唯一**的实时通道；**不是事件总线，没有事件类型包装**。

### 5.1 连接

`# UAT（连接 TurboFlow WS 服务端；鉴权见 §7.1） wss://api.turboflow-test.xyz/api/v1/vendor/feed  # 凭据走 Header（推荐）：API-KEY / SIGN / TIMESTAMP # WS 客户端不便带 Header 时回退 query： wss://api.turboflow-test.xyz/api/v1/vendor/feed?api_key=<pubkey>&sign=<sig>&timestamp=<unix_sec>`

- 协议：WebSocket（wss://，TLS 1.2+）
    
- 鉴权：Ed25519 签名，见 §7.1（API-KEY / SIGN / TIMESTAMP）；**不再使用** ?token=
    
- 心跳：双方互发 ping / pong（30s 周期）；60s 无任何上行帧 → 断开
    
- 重连：连接方指数退避（1s/2s/4s/8s 封顶 30s）
    

**连接首帧（market_info 快照）**：连接建立后，TurboFlow 立即下发一帧当前可用盘口快照。 每个 markets[] 元素对应 **一个 Polymarket orderbook**（带 slug），不是 event 级聚合； SIG 按 slug 定价、推赔率。

`{   "op": "market_info",   "at_ms": 1748764830123,   "events": [     {       "league_code": "FIF",       "match_id": "1001",       "event_slug": "fif-egy-rus-2026-05-28",       "polymarket_event_slug": "fif-egy-rus-2026-05-28",       "sig_match_id": "sig:fif:2026-05-28:egy-rus",       "status": "incoming",       "home_team_slug": "egy",       "away_team_slug": "rus",       "kickoff_at": "2026-05-28T19:30:00Z",       "market_groups": [         {           "market_code": "result_1x2",           "market_name": "Full Time Result",           "markets": [             { "outcome_key": "home_win", "outcome_label": "Egypt", "side": "yes", "slug": "fif-egy-rus-2026-05-28-egy" },             { "outcome_key": "draw",     "outcome_label": "Draw",  "side": "yes", "slug": "fif-egy-rus-2026-05-28-draw" },             { "outcome_key": "away_win", "outcome_label": "Russia","side": "yes", "slug": "fif-egy-rus-2026-05-28-rus" }           ]         },         {           "market_code": "result_ht",           "market_name": "Half Time Result",           "markets": [             { "outcome_key": "home_win_ht", "side": "yes", "slug": "fif-egy-rus-2026-05-28-halftime-result-home" },             { "outcome_key": "draw_ht",     "side": "yes", "slug": "fif-egy-rus-2026-05-28-halftime-result-draw" },             { "outcome_key": "away_win_ht", "side": "yes", "slug": "fif-egy-rus-2026-05-28-halftime-result-away" }           ]         }       ]     }   ] }`

**Slug 规则**（与 Polymarket 一致，event_slug 取自已确认的 polymarket_event_link）：

|   |   |
|---|---|
|层级|规则|
|event_slug|<league>-<home>-<away>-<yyyy-mm-dd>|
|status|incoming=未开赛；open=进行中（Live / Halftime）|
|result_1x2|home_win → <event_slug>-<home>；draw → <event_slug>-draw；away_win → <event_slug>-<away>|
|result_ht|home_win_ht → <event_slug>-halftime-result-home；draw_ht → ...-draw；away_win_ht → ...-away|

仅对已链接 Polymarket event 的比赛优先使用 confirmed slug；未链接时按内部 match 元数据退化拼 event_slug（规则同上），league_code 取 match_league.short_name。

event_slug **与** polymarket_event_slug**（供 SIG 精确匹配）**：

|   |   |   |
|---|---|---|
|字段|必出现|含义|
|event_slug|是|内部路由 slug。已链接 Polymarket 时即 confirmed slug；未链接时按 match 元数据退化拼，且 <league> 前缀按已知映射归一到 Polymarket 前缀（如 wc→fifwc、intl→fif）|
|polymarket_event_slug|是（值可空）|仅当我方已确认该盘口绑定 Polymarket 时回填其**原始 slug**，供 SIG 精确匹配；为空表示我方未确认该盘口在 Polymarket 存在|

SIG 匹配建议：polymarket_event_slug 非空时按它精确匹配（权威）；为空时回落 event_slug（前缀已尽量对齐 Polymarket）。内部 short_name → poly 前缀 映射当前确认：WC→fifwc、INTL→fif，前缀本就一致的联赛无需登记。

同款结构也可随时用 GET /api/v1/vendor/markets 主动查询（响应 { "events": [...] }）。

### 5.2 订阅

TurboFlow 连接后发起一次订阅，SIG 持续推送：

`{ "op": "sub", "league_codes": ["EPL", "UCL", "WC"] }`

|   |   |   |
|---|---|---|
|字段|必填|说明|
|op|是|固定 sub；未来扩展 unsub / replace|
|league_codes|是|订阅的系列赛 code 列表；空数组表示全订阅|

### 5.3 赔率数据帧（核心）

SIG 主动推送，**单帧 = 单个 outcome 的一次赔率快照**（v3.4 起；一次只返一个 outcome，不再用 outcomes 数组）。**无** type **/** event_id **包装；TurboFlow 用** (league_code, sig_match_id, market_code, outcome_key, at_ms) **去重 + 排序**。

`{   "league_code": "EPL",   "sig_match_id": "sig-20260601-arsenal-chelsea",   "polymarket_event_id": "0x9a3f...e21",   "scope": "match",   "kickoff_at": "2026-06-01T19:00:00Z",   "market_code": "result_1x2",   "market_status": "open",   "outcome": {     "outcome_key": "home_win",     "buy_decimal_odds": "2.150000",     "sell_decimal_odds": "1.950000",     "status": "active"   },   "at_ms": 1748764830123 }`

> 同一市场的多个 outcome（如 home_win / draw / away_win）拆成多帧分别推送，每帧一个 outcome。
> 
> **兼容**：旧版 outcomes 数组帧仍被接收（REST 兜底快照可整市场返回数组），但 WS 实时推送请用单 outcome。

#### 字段说明

|   |   |   |
|---|---|---|
|字段|必填|说明|
|league_code|是|必须取自 §2 表|
|sig_match_id|scope=match 必填|scope=series 时仍建议填关联比赛 ID，可空|
|polymarket_event_id|否|SIG 侧对应的 Polymarket event id；仅用于对账透传，**不参与定位**（定位仍用四元组）。SIG 有则带、无则省|
|scope|是|match / series|
|kickoff_at|scope=match 必填|开赛时间|
|market_code|是|必须取自 §4 表|
|market_status|是|open / closed（关盘，不再接受投注）|
|outcome|是|单个 outcome 的最新赔率快照；字段按 §4.3|
|at_ms|是|SIG 生产此快照的时间戳（UTC 毫秒）|

#### scope=series 示例

`{   "league_code": "WC",   "scope": "series",   "market_code": "series_champion",   "candidate_id": "team_argentina",   "candidate_label": "Argentina",   "market_status": "open",   "outcome": {     "outcome_key": "yes",     "buy_decimal_odds": "5.200000",     "sell_decimal_odds": "4.900000",     "status": "active"   },   "at_ms": 1748764830123 }`

|   |   |
|---|---|
|额外字段|说明|
|candidate_id|scope=series 必填；如球队 / 球员 ID|
|candidate_label|scope=series 必填；展示文案|

#### 关盘示例

关盘时 market_status=closed，TurboFlow 会把该市场下所有 outcome 一次性置 suspended（无需逐 outcome 推）。 outcome 仍可携带最后一刻赔率（仅展示）：

`{   "league_code": "EPL",   "sig_match_id": "sig-20260601-arsenal-chelsea",   "scope": "match",   "market_code": "result_1x2",   "market_status": "closed",   "outcome": {     "outcome_key": "home_win",     "buy_decimal_odds": "1.910000",     "sell_decimal_odds": "1.750000",     "status": "suspended"   },   "at_ms": 1748767200000 }`

### 5.4 推送频率

- 赔率变化即推；同时 SIG 应保证**每 outcome 至少 5s 一帧**（即使无变化，作为活跃心跳）
    
- 一帧只含一个 outcome；同一市场的多个 outcome 拆多帧推送
    
- 历史 / 重发不允许（不补推过去的旧帧）；TurboFlow 用 at_ms 严格判定丢弃旧帧
    

### 5.5 心跳

`// SIG → TurboFlow { "op": "ping", "at_ms": 1748767200000 }  // TurboFlow → SIG { "op": "pong", "at_ms": 1748767200005 }`

任意一端 60s 无对端帧即断开重连。

## 7. 鉴权与限流

### 7.1 凭据（Ed25519 签名，与主站统一）

鉴权方式统一到 TurboFlow 主站 API（turboflow-api-doc signature-generation）：**Ed25519 非对称签名**。 vendor 持私钥，TurboFlow 存公钥（vendor_api_keys.api_key 列存 64 字符 hex 公钥）。

请求头（WS 客户端不便带 header 时可回退同名 query api_key / sign / timestamp）：

|   |   |
|---|---|
|Header|说明|
|API-KEY|vendor 的 Ed25519 公钥（64 字符 hex）|
|SIGN|请求签名（hex）|
|TIMESTAMP|Unix 秒级时间戳，容差 ±300s（防重放）|

签名串与主站一致（vendor feed 端点均为 GET、无 body，path 取**纯 URL 路径**不含 query）：

`method=GET&path={path}&timestamp={timestamp}&access-key={apiKey}`

签名步骤：

1. 构造上面的签名串
    
2. 用公钥(hex 解码后)对签名串做 HMAC-SHA256，得到摘要
    
3. 用 vendor Ed25519 私钥对摘要签名
    
4. 签名结果转 hex 放入 SIGN
    

约定：

- 按环境隔离：dev / sit / uat / prod 各一套密钥对
    
- 公钥由运维写入 vendor_api_keys，enabled=false 即时失效（最长 60s reload）
    
- 私钥由 vendor 自持，**不进 git / yaml / 任何配置**
    
- 可用 scripts/genvendorkey 生成密钥对（公钥入库、私钥发 vendor）
    

### 7.2 限流

|   |   |
|---|---|
|接口|默认|
|GET /v1/odds/snapshot|10 QPS（仅启动 / 补齐用）|
|GET /v1/odds/match|30 QPS|
|WebSocket 连接数|单 API Key ≤ 4 条|

超限 REST 返回 429 + Retry-After；WS 超连接数直接拒连。
