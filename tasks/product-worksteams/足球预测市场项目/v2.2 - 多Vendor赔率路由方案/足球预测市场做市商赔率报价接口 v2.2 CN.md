## 5. WebSocket（实时赔率数据流）

SIG 通过 WebSocket **持续推送赔率数据帧**。这是**唯一**的实时通道；**不是事件总线，没有事件类型包装**。

### 5.1 连接

- **API_KEY ：**  
    **7f457c3bb3940ed42f665a1135ed2a21cc41191d2dd4e66aada00fb40030edf4**
    
- **PRIVATE_SEED (私钥种子，作 Ed25519 私钥)：**  
    **05285997edac7c463232bf1bb02ba8a26589bb4ce9e1d86b998a88a20d97caad**
    
- 协议：WebSocket（wss://，TLS 1.2+）
    
- 鉴权：Ed25519 签名，见 §7.1（API-KEY / SIGN / TIMESTAMP）；**不再使用** ?token=

### 5.3 赔率数据帧（核心）

SIG 主动推送，**单帧 = 单市场的一次赔率快照**。**无** type **/** event_id **包装；TurboFlow 用** (league_code, sig_match_id, market_code, at_ms) **去重 + 排序**。

`{   "league_code": "EPL",   "sig_match_id": "sig-20260601-arsenal-chelsea",   "scope": "match",   "kickoff_at": "2026-06-01T19:00:00Z",   "market_code": "result_1x2",   "market_status": "open",   "outcomes": [     { "outcome_key": "home_win", "buy_decimal_odds": "2.150000", "sell_decimal_odds": "1.950000", "status": "active" },     { "outcome_key": "draw",     "buy_decimal_odds": "3.500000", "sell_decimal_odds": "3.300000", "status": "active" },     { "outcome_key": "away_win", "buy_decimal_odds": "4.000000", "sell_decimal_odds": "3.800000", "status": "active" }   ],   "at_ms": 1748764830123 }`

#### 字段说明

|   |   |   |
|---|---|---|
|字段|必填|说明|
|league_code|是|必须取自 §2 表|
|sig_match_id|scope=match 必填|scope=series 时仍建议填关联比赛 ID，可空|
|scope|是|match / series|
|kickoff_at|scope=match 必填|开赛时间|
|market_code|是|必须取自 §4 表|
|market_status|是|open / closed（关盘，不再接受投注）|
|outcomes|是|该市场所有 outcome 的最新赔率快照；按 §4.3|
|at_ms|是|SIG 生产此快照的时间戳（UTC 毫秒）|

#### scope=series 示例

`{   "league_code": "WC",   "scope": "series",   "market_code": "series_champion",   "candidate_id": "team_argentina",   "candidate_label": "Argentina",   "market_status": "open",   "outcomes": [     { "outcome_key": "yes", "decimal_odds": "5.200000", "status": "active" },     { "outcome_key": "no",  "decimal_odds": "1.240000", "status": "active" }   ],   "at_ms": 1748764830123 }`

|   |   |
|---|---|
|额外字段|说明|
|candidate_id|scope=series 必填；如球队 / 球员 ID|
|candidate_label|scope=series 必填；展示文案|

#### 关盘示例

关盘时 market_status=closed，outcomes 仍可携带最后一刻赔率（仅展示）：

`{   "league_code": "EPL",   "sig_match_id": "sig-20260601-arsenal-chelsea",   "scope": "match",   "market_code": "result_1x2",   "market_status": "closed",   "outcomes": [     { "outcome_key": "home_win", "decimal_odds": "1.910000", "status": "suspended" },     { "outcome_key": "draw",     "decimal_odds": "3.700000", "status": "suspended" },     { "outcome_key": "away_win", "decimal_odds": "4.400000", "status": "suspended" }   ],   "at_ms": 1748767200000 }`

### 5.4 推送频率

- 赔率变化即推；同时 SIG 应保证**每市场至少 5s 一帧**（即使无变化，作为活跃心跳）
    
- 同一 (league_code, sig_match_id, market_code) 不允许在一帧内分多包；必须整市场快照
    
- 历史 / 重发不允许（不补推过去的旧帧）；TurboFlow 用 at_ms 严格判定丢弃旧帧
    

### 5.5 心跳

`// SIG → TurboFlow { "op": "ping", "at_ms": 1748767200000 }  // TurboFlow → SIG { "op": "pong", "at_ms": 1748767200005 }`

任意一端 60s 无对端帧即断开重连。

## 6. REST（启动同步 / 断线补齐）

REST 仅作为兜底，**不是实时通道**；正常运行只用 WebSocket。

### 6.1 全量赔率（按系列赛）

`GET /v1/odds/snapshot?league_code=EPL`

**响应**：与 WebSocket 数据帧格式一致，外层包数组：

`{   "league_code": "EPL",   "items": [     { "sig_match_id": "...", "scope": "match", "market_code": "result_1x2", "market_status": "open", "outcomes": [...], "at_ms": 1748764830123 },     { "sig_match_id": "...", "scope": "match", "market_code": "btts",        "market_status": "open", "outcomes": [...], "at_ms": 1748764830200 }   ] }`

**用途**：服务启动 / WebSocket 长时间断开后调一次拉全量。

`GET /v1/odds/match?league_code=EPL&sig_match_id=sig-20260601-arsenal-chelsea`

响应同上，items[] 仅该比赛的全部市场。

`{ "code": "INVALID_LEAGUE_CODE", "message": "league_code not in spec table" }`

|   |   |
|---|---|
|code|含义|

## 7. 鉴权与限流

### 7.1 凭据（Ed25519 签名，与主站统一）

鉴权方式统一到 TurboFlow 主站 API（turboflow-api-doc signature-generation）：**Ed25519 非对称签名**。 vendor 持私钥，TurboFlow 存公钥（vendor_api_keys.api_key 列存 64 字符 hex 公钥）。

请求头（WS 客户端不便带 header 时可回退同名 query api_key / sign / timestamp）：

|   |   |
|---|---|
|Header|说明|
|API-KEY|vendor 的 Ed25519 公钥（64 字符 hex）|
|SIGN|请求签名（hex）|
|TIMESTAMP|Unix 秒级时间戳，容差 ±300s（防重放）|

签名串与主站一致（vendor feed 端点均为 GET、无 body，path 取**纯 URL 路径**不含 query）：

`method=GET&path={path}&timestamp={timestamp}&access-key={apiKey}`

签名步骤：

1. 构造上面的签名串
    
2. 用公钥(hex 解码后)对签名串做 HMAC-SHA256，得到摘要
    
3. 用 vendor Ed25519 私钥对摘要签名
    
4. 签名结果转 hex 放入 SIGN
    

约定：

- 按环境隔离：dev / sit / uat / prod 各一套密钥对
    
- 公钥由运维写入 vendor_api_keys，enabled=false 即时失效（最长 60s reload）
    
- 私钥由 vendor 自持，**不进 git / yaml / 任何配置**
    
- 可用 scripts/genvendorkey 生成密钥对（公钥入库、私钥发 vendor）
    

### 7.2 限流

|   |   |
|---|---|
|接口|默认|
|GET /v1/odds/snapshot|10 QPS（仅启动 / 补齐用）|
|GET /v1/odds/match|30 QPS|
|WebSocket 连接数|单 API Key ≤ 4 条|

超限 REST 返回 429 + Retry-After；WS 超连接数直接拒连。

- 订阅 league_codes: ["EPL"] 后能持续收到该联赛全部比赛 + 全部市场的赔率帧

- 关盘时 market_status=closed
    
- 断网 30s 后重连 → 重新订阅后能收到新帧
    
- ping/pong 心跳正常
    

REST：

- GET /v1/odds/snapshot?league_code=EPL 拉到全部比赛全部市场
    
- GET /v1/odds/match 拉到单场比赛全市场
    
- 用未注册 league_code 请求 → 返回 INVALID_LEAGUE_CODE
    
- 故意发限流 → 返回 429 + Retry-After
    

通用：

- 同一 sig_match_id 在不同帧中稳定
    
- at_ms 单调递增（同市场范围内）
    
- 所有 league_code 取自 §2，所有 market_code / outcome_key 取自 §4
