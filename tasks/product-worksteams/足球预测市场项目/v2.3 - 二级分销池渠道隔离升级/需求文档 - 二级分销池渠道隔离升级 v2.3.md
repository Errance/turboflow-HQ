# 需求文档：足球预测市场二级分销池渠道隔离升级（v2.3）

**状态：** Draft  
**主 PRD：** [../../二级分销池/PRD-二级分销池-升级方案.md](../../二级分销池/PRD-二级分销池-升级方案.md)  

## 目标

足球预测市场 v2.3 需要支持渠道站独立运营入口，并实现渠道身份隔离、资金隔离、报价 markup、分销池路由开关、渠道收入与池子盈亏对账。

## 范围

- 渠道域名进入足球预测市场时，后端识别 `channel_id`。
- 渠道用户注册 / 登录进入独立 `identity_domain`，不可跨登录 TF 主站。
- 渠道用户资产进入独立 `fund_domain`，不可与 TF 主站余额跨充值、提现或交易复用。
- 足球预测市场用户端报价应用渠道 markup。
- 下单服务端权威计算 markup，确保展示价、下单价、结算价一致。
- 足球预测市场支持分销池路由开关：
  - OFF：进入 TF 公共池。
  - ON：进入渠道分销池。
- 订单、cashbook、settlement 记录保留渠道字段、资金域字段与 `pool_id`，用于渠道收入与池子盈亏对账。

## 非范围

- 不做同一笔订单 L1:L2 比例拆分。
- 不做跨站统一登录。
- 不做跨资金域余额划转。
- 不把多 Vendor best_quote 作为 v2.3 阻塞依赖，但 v2.3 需要兼容未来 best_quote。

## 验收要点

- 渠道站足球预测订单能记录 `channel_id`、`identity_domain`、`fund_domain`、`pool_id`。
- 渠道用户不能直接登录 TF 主站，TF 主站用户不能直接登录渠道站。
- 渠道用户的用户名前缀使用channel_id，而不是"Turbo_"。
- 渠道余额不能提现到 TF 主站出金路径，TF 主站余额不能提现到渠道出金路径。
- 渠道报价包含 markup，且展示价、下单价、结算价一致。
- 分销池开关 OFF 时订单进入 TF 公共池。
- 分销池开关 ON 且渠道池健康时订单进入渠道分销池。
- markup 收入与渠道分销池盈亏可按渠道对账。
