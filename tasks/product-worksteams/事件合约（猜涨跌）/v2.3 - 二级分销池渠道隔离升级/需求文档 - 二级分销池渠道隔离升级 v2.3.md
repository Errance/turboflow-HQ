# 需求文档：事件合约二级分销池渠道隔离升级（v2.3）

**状态：** Draft  
**主 PRD：** [../../二级分销池/PRD-二级分销池-升级方案.md](../../二级分销池/PRD-二级分销池-升级方案.md)  

## 目标

事件合约 v2.3 需要支持渠道站独立运营入口，并在不改变事件合约公共池结算模式的前提下，实现渠道身份隔离、资金隔离、报价 markup 与渠道收入对账。

## 范围

- 渠道域名进入事件合约时，后端识别 `channel_id`。
- 渠道用户注册 / 登录进入独立 `identity_domain`，不可跨登录 TF 主站。
- 渠道用户资产进入独立 `fund_domain`，不可与 TF 主站余额跨充值、提现或交易复用。
- 事件合约用户端报价应用渠道 markup。
- 下单服务端权威计算 markup，确保展示价、下单价、结算价一致。
- 事件合约渠道订单继续 100% 进入 TF 公共池。
- 订单、cashbook、settlement 记录保留渠道字段，用于渠道收入对账。

## 非范围

- 不启用事件合约渠道分销池。
- 不做 L1:L2 比例分流。
- 不做跨站统一登录。
- 不做跨资金域余额划转。

## 验收要点

- 渠道站事件合约订单能记录 `channel_id`、`identity_domain`、`fund_domain`。
- 渠道用户不能直接登录 TF 主站，TF 主站用户不能直接登录渠道站。
- 渠道余额不能提现到 TF 主站出金路径，TF 主站余额不能提现到渠道出金路径。
- 渠道报价包含 markup，且展示价、下单价、结算价一致。
- 事件合约渠道订单始终进入 TF 公共池。
- markup 收入可按渠道对账。
